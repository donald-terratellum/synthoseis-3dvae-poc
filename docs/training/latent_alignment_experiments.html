Latent Alignment Training Guide

Goals, ownership, implementation, theory, references, and ablation workflow for improving reconstruction detail.

training VAE latent losses ablation

Goals

  • Improve reconstruction detail in predictions, especially fault-like discontinuities and localized vertical offsets.
  • Reduce latent mismatch between input, prediction, and label in representative plots.
  • Bias optimization toward stronger encoder learning and relatively weaker decoder dynamics.

Who

  • Primary audience: researchers and engineers training the 3D VAE in this repository.
  • Secondary audience: reviewers evaluating reproducibility and methodology.

What

This guide documents two optional training modifications:

  1. Latent consistency losses:
    • prediction vs label latent alignment
    • prediction vs input latent alignment
  2. Encoder/decoder learning-rate multipliers:
    • increase encoder LR relative to base LR
    • decrease decoder LR relative to base LR

When

  • Use when reconstructions appear over-smoothed and miss structure visible in labels.
  • Use when representative latent plots show large separation between prediction and label.
  • Do not treat this as a replacement for data quality and augmentation review.

How

New flags

  • --latent_pred_target_weight
  • --latent_pred_input_weight
  • --latent_alignment_detach_targets / --no_latent_alignment_detach_targets
  • --encoder_lr_mult
  • --decoder_lr_mult

Meaning

  • --latent_pred_target_weight: aligns encoder(prediction) to encoder(label).
  • --latent_pred_input_weight: aligns encoder(prediction) to encoder(input).
  • --latent_alignment_detach_targets (default): detaches target branches for stability.
  • --encoder_lr_mult > 1.0: stronger encoder optimization pressure.
  • --decoder_lr_mult < 1.0: relatively weaker decoder optimization dynamics.

Objective

L = L_recon + beta * L_KL + lambda_pt * ||E(y_hat)-E(y)||^2 + lambda_pi * ||E(y_hat)-E(x)||^2 + lambda_gan * L_gan

Where x is input, y_hat is prediction, y is label, and E is encoder mean branch.

Usage

Baseline

uv run python scripts/train.py \
  --data data/train_32-32-64.zarr \
  --validation_data data/val_32-32-64.zarr \
  --batch_size 50 \
  --number_batches 100 \
  --epochs 150 \
  --augment \
  --learning_rate 1e-4 \
  --weight_decay 1e-4 \
  --out_dir checkpoints_baseline

Latent alignment only

uv run python scripts/train.py \

Optional sensitivity checks

  • A4 Higher latent pressure: latent_pred_target_weight=0.10
  • A5 Add input alignment: latent_pred_input_weight=0.02

Readouts

  • Validation loss trajectory.
  • Representative plots at fixed examples over epochs.
  • Visual preservation of fault discontinuities.
  • TensorBoard: train/latent_pred_target_loss, train/latent_pred_input_loss, train/encoder_lr, train/decoder_lr.

Decision criteria

  • Promote if fault detail consistently improves and training remains stable.
  • Reject or down-weight if instability increases or validation trends regress materially.

Theory And Rationale

  • Pointwise losses can produce over-smoothed reconstructions under uncertainty.
  • Feature-space or latent-space matching can improve structure preservation.
  • Denoising formulations align with corrupted-input to clean-target training.
This approach is an empirical training strategy and should be validated with controlled ablations before broad adoption.
