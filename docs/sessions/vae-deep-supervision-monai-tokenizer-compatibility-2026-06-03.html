Session Summary: VAE Deep Supervision (MONAI-Style) With Tokenizer Compatibility (2026-06-03)

Context And Goals

This session closed out the deep-supervision implementation plan captured in docs/plans/deep_supervision_feature_ala_monai.md.

Primary goals were to improve decoder supervision for coarse/fine structure, keep latent outputs stable for tokenizer workflows, keep changes optional/backward-compatible, and validate behavior with focused tests.

What Was Done

  • Added optional deep-supervision support to the VAE decoder in src/model.py.
  • Added auxiliary 1x1x1 decoder prediction heads at coarse and mid stages.
  • Upsampled auxiliary predictions to full output resolution and exposed them only through an explicit deep-supervision return path.
  • Added DeepSupervisionLoss in src/deep_supervision.py with weighted multi-scale aggregation.
  • Integrated deep-supervision training/validation handling in scripts/train.py.
  • Added CLI options --deep_supervision and --deep_supervision_weights.
  • Removed latent alignment loss wiring from this training flow and replaced it with deep-supervision reconstruction handling.
  • Updated checkpoint metadata and load compatibility logic for optional auxiliary heads.
  • Updated tokenizer adapter loading compatibility in src/tokenizer/core/model_adapter.py.
  • Updated test arg fixtures in tests/test_input_augmentations.py.
  • Added tests/test_deep_supervision.py for shape checks, weighted loss validation, inference contract, and gradient-flow checks.

How Was It Done

  • Refactored decoder internals from a single sequential block into explicit staged modules to tap intermediate features cleanly.
  • Added optional auxiliary heads and scale-up logic with trilinear interpolation.
  • Created a reusable weighted deep-supervision loss wrapper around nn.MSELoss.
  • Added conditional train/validation branches to request and consume deep-supervision outputs.
  • Used strict-but-compatible checkpoint loading (strict=False + allowlists) to protect backward compatibility.
