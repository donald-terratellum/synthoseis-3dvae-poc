Session Summary: Augmentation Pipeline Updates (2026-06-03)

Context And Goals

The session focused on extending and stabilizing the training input augmentation pipeline for 3D VAE training.

Primary goals were to add sparse-input augmentation options, enforce transform exclusivity, replace the slow Mitchell-like sparse selector, align training and validation controls, diagnose representative-plot artifacts, and update defaults/tests.

Command context included runs with --input_extrema_prob 0.0 --input_sparse_keep_prob 1.0 --input_decimate_trilinear_prob 0.0 targeting checkpoints_poisson_disc.

What Was Done

  • Added one-of-three input transform selection (extrema / sparse / decimate) using weighted probabilities.
  • Implemented sparse methods: Poisson-like and uniform-threshold with random 50/50 selection.
  • Removed Mitchell-like sparse method from active pipeline and CLI.
  • Added decimate+trilinear input augmentation.
  • Synced validation input transform behavior with training transform family/weights when enabled.
  • Diagnosed all-zero representative training input artifacts and fixed sparse assignment for non-contiguous arrays.
  • Updated sparse default keep range from [0.10, 0.20] to [0.10, 0.30].
  • Updated vertical warp defaults to min_step=0.5 and max_step=2.0.
  • Added/updated benchmark and unit tests; generated benchmark PNG artifacts.

How It Was Done

  • Modified augmentation and training data pipeline code in src/augmentations.py and scripts/train.py.
  • Added/updated tests in tests/test_input_augmentations.py, tests/test_sparse_keep_vs_uniform_benchmark.py, and tests/test_vertical_warp.py.
  • Inspected representative metadata and plots from checkpoints_poisson_disc to reproduce and diagnose behavior.
  • Validated changes with targeted unittest runs and CLI smoke checks.
